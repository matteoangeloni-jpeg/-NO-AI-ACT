Come citare NO AI ACT

Se usi NO AI ACT in una tesi, un articolo, una sperimentazione didattica o un lavoro derivato, questa pagina fornisce i formati di citazione corretti. Indica sempre la versione esatta che hai usato: il gioco evolve e i contenuti dei casi possono cambiare tra versioni.

01Citazione in formato APA

Formato APA (7ª edizione) per software:

Angeloni, M. (2026). NO AI ACT — Simulatore di una società non regolata (Versione 2.0.0) [Software]. https://www.no-ai-act.eu/

Sostituisci "2.0.0" con la versione effettivamente usata: la trovi nel piè di pagina del sito, nella schermata del titolo del gioco e nel file package.json del repository.

02BibTeX

@software{angeloni_no_ai_act_2026,
  author  = {Angeloni, Matteo},
  title   = {NO AI ACT --- Simulatore di una societ{\`a} non regolata},
  year    = {2026},
  version = {2.0.0},
  url     = {https://www.no-ai-act.eu/},
  note    = {Codice sorgente: https://github.com/matteoangeloni-jpeg/-NO-AI-ACT.
             Codice MIT; contenuti educativi CC BY 4.0.}
}

03Perché citare la versione esatta

  • Riproducibilità: casi, testi e regole di punteggio sono fissati per versione. Chi replica uno studio deve poter recuperare esattamente ciò che i partecipanti hanno visto.
  • Dove trovarla: piè di pagina del sito, package.json, tag di release su GitHub Releases.
  • Archivio storico: ogni release taggata su GitHub conserva il codice e i contenuti di quella versione.

04DOI

Un DOI (Digital Object Identifier) non è ancora disponibile. È prevista l'archiviazione del progetto su Zenodo in occasione della release 2.0: quando il DOI sarà attivo verrà pubblicato qui, nel file CITATION.cff e nel README del repository. Fino ad allora, cita l'URL canonico e la versione.

05Il file CITATION.cff

Il repository contiene un file CITATION.cff in formato Citation File Format: GitHub lo legge automaticamente e mostra il pulsante "Cite this repository", e gli strumenti di gestione bibliografica possono importarlo direttamente.
